提交代码之前的事

四个区域

git 通过四个区域来分层控制代码，因此从编写代码到将代码提交到远程仓库，需要经过这四个步骤。

  1. 工作区(working directory)：被修改的文件(Modified files)，被删除的文件(Deleted files)，未跟踪文件(Untracked files) 处于该区。
  2. 缓存区(Stage)：Staged files 处于该区。
  3. 本地仓库(Local Repository)：Committed files 处于该区。
  4. 远程仓库(Remote Repository)：Pushed files 处于该区。

四个区域之间提交代码

git add 工作区 ——> 缓存区
git commit -m “Description” 缓存区 ——> 本地仓库
git pull 远程仓库 ——> 本地仓库 (执行 push 之前，确保首先执行 pull，该过程可能产生冲突（conflict）)
git push 本地仓库 ——> 远程仓库

四个区域之间比较代码

git diff 查看工作区和暂存区之间的代码差异。
git diff –cached 查看暂存区和本地仓库之间的代码差异。
git diff master origin/master 查看本地仓库和远程仓库之间的代码差异。
Note: 命令后面可以指定需要比较的文件名。

在Git工作区的根目录下创建一个特殊的.gitignore文件，然后把要忽略的文件名填进去，Git就会自动忽略这些文件。
忽略文件的原则是：
忽略操作系统自动生成的文件，比如缩略图等；
忽略编译生成的中间文件、可执行文件等，也就是如果一个文件是通过另一个文件自动生成的，那自动生成的文件就没必要放进版本库，比如Java编译产生的.class文件；
忽略你自己的带有敏感信息的配置文件，比如存放口令的配置文件。

git reset 和 git revert 的区别
git revert 比 git reset 更安全，git reset 撤销提交不会留下痕迹，撤销的东西无法挽回，而git revert在将要撤销的部分作为一次新的提交，因此是可以挽回的。

git stash 用于保存和恢复工作进度

git stash

保存当前的工作进度。会分别对暂存区和工作区的状态进行保存，但不会对 Untracked files 做任何保存或其他操作。

git stash save “message…”

这条命令实际上是第一条 git stash 命令的完整版

git stash list

显示进度列表。此命令显然暗示了git stash 可以多次保存工作进度，并用在恢复时候进行选择

git stash pop [–index] []

如果不使用任何参数，会恢复最新保存的工作进度，并将恢复的工作进度从存储的工作进度列表中清除。

如果提供参数（来自 git stash list 显示的列表），则从该 中恢复。恢复完毕也将从进度列表中删除 。

选项–index 除了恢复工作区的文件外，还尝试恢复暂存区。

git stash apply [–index] []

除了不删除恢复的进度之外，其余和 git stash pop 命令一样

Linux find and grep command

find 用来查找文件或者目录
grep 用来查找所有匹配正则表达式的地方，grep 要比 find 查找的粒度更细。
参考：
https://www.linode.com/docs/tools-reference/tools/find-files-in-linux-using-the-command-line/

find 命令

find [-H] [-L] [-P] [-D debugopts] [-Olevel] [path…] [expression]

-name 根据文件/目录名查找

find path -name “filename”
find / -name “filename” #根目录下查找指定文件或者目录

-mtime 根据文件修改时间查找

find path -mtime -n/+n

-type 根据文件类型查找

find path -type []

-size 根据文件大小查找

find path -size

-perm 根据文件访问权限查找
-user 根据文件属主查找
-group 根据文件所属的组查找

-exec 和 xargs 用于对所匹配到的文件执行其他命令。

grep 命令

grep -rn “string” path

-n, –line-number
Each output line is preceded by its relative line number in the file, starting at line 1. The line number counter is reset for each file processed. This option is ignored if -c, -L, -l, or -q is specified.

-R, -r, –recursive
Recursively search subdirectories listed.

-v, –invert-match
Selected lines are those not matching any of the specified patterns.
